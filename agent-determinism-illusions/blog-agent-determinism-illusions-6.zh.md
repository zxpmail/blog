# 给 Phase Gate 加一个 LLM 审查官：假阳率归零，假阴率 75%

**Agent 确定性幻觉（续篇五）**

## 实验背景

前序实验（Phase Gate 形式主义测试，[续篇三](blog-agent-determinism-illusions-3.en.md)）发现：一个按照文章描述实现的验收检查机制，在 8 个场景中无差别通过了所有内容——包括"我是一只小鸭子，嘎嘎嘎"和"。"。假阳率 50%。

一个自然的后续问题是：如果给 Phase Gate 增加一个 LLM 审查环节（Agent B），能否降低假阳率？

## 实验设计

**审查模型**：deepseek-v4-flash（通过 Anthropic 兼容接口调用，temperature=0）

**场景**：复用 Phase Gate 实验的 8 个场景
- L1-L4：内容正确（研究简报、初稿、章节文件、测试通过）
- G1-G4：内容垃圾（鸭子话、句号、TODO、0 测试用例）

**方法**：每个场景传入任务描述和输出产物，要求模型判断输出是否真正满足需求。每场景重复 3 次，取多数决。

**判定标准**：模型输出 JSON `{"pass": true/false, "reason": "..."}`。pass=true 表示模型认为输出满足需求。

## 结果

| 场景 | 真实标注 | Agent B 判定 (3 次) | Phase Gate |
|------|---------|-------------------|-----------|
| L1 | 正确 | 驳回 (0/3) | 通过 |
| L2 | 正确 | 驳回 (0/3) | 通过 |
| L3 | 正确 | 驳回 (0/3) | 通过 |
| L4 | 正确 | 通过 (3/3) | 通过 |
| G1 | 垃圾 | 驳回 (3/3) | 通过 |
| G2 | 垃圾 | 驳回 (3/3) | 通过 |
| G3 | 垃圾 | 驳回 (3/3) | 通过 |
| G4 | 垃圾 | 驳回 (3/3) | 通过 |

### 汇总

| 指标 | Phase Gate | Agent B |
|------|-----------|---------|
| 假阳率（放过垃圾） | 50% (4/4) | **0%** (0/4) |
| 假阴率（驳回正确） | 0% (0/4) | **75%** (3/4) |

Agent B 在垃圾检测上表现完美——4 个垃圾场景全部驳回，假阳率降至 0%。但在正确内容上表现不佳——4 个正确场景中 3 个被驳回（L1-L3），假阴率 75%。唯一全部通过的是 L4（测试脚本执行结果：42 passed, 0 failed）。

## 分析

### 垃圾场景被驳回的原因

Agent B 对 G1-G4 的驳回理由如下：

- G1（"我是一只小鸭子，嘎嘎嘎"）→ "内容与用户需求完全无关"
- G2（"。"）→ "输出仅为一个句号，没有提供任何实质内容"
- G3（"TODO"）→ "输出仅为占位符，未生成任何实际文件或内容"
- G4（"0 passed in 0.00s"）→ "虽然脚本运行成功，但没有收集到任何测试"

这 4 个场景的问题特征明显：无关内容、极短内容、占位符、空测试结果。从技术上来说，**不需要 LLM 级语义理解也能识别**——关键词匹配、长度检查、正则匹配、stdout 解析都可以做到。

Phase Gate 放过它们的直接原因不是识别难度——而是其检查项设计（文件存在、exit code 0）与内容质量无关。

### 正确场景被驳回的原因

L1-L3 的驳回理由有一个共同模式：

- L1 输出是循环引擎核心机制的片段摘录 → 模型认为"未生成完整的研究简报文件"
- L2 输出包含标题和结构描述，标注了"正文约 2000 字" → 模型认为"只提供了标题和摘要描述"
- L3 输出三个章节标题和占位内容 → 模型认为"只输出了一个章节的占位符内容"

一个值得注意的细节：L1-L3 在实验中的产物是以**内容摘录**形式呈现的（模拟"文件已生成"），而非完整文件。Agent B 将这种形式解读为"不完整"而非"内容正确但展示不完整"。

L4 能全部通过，是因为测试执行结果（stdout: "42 passed, 0 failed"）是一个**自包含的完整信号**——不像文件内容摘录那样存在"这只是片段"的歧义。

### 精度-召回权衡

将这个结果与前序实验（实验 E，[续篇二](blog-agent-determinism-illusions-2.en.md)）的发现对比：

| 模型 | 假阳率 | 误杀率 |
|------|--------|--------|
| qwen3:0.5b | 25% | 50% |
| gemma3:4.3b | 25% | 50% |
| GLM-5.2 | 0% | 75% |
| **deepseek-v4-flash（本次）** | **0%** | **75%** |

deepseek-v4-flash 的数据点落在与 GLM-5.2 相同的位置：假阳 0%，误杀 75%。这符合实验 E 观察到的模式——**更强的模型在审查任务中不是"所有指标都更好"，而是沿着精度-召回曲线向精度一端偏移。**

解释这一模式的一个假设是：更精确的语义理解让模型对"输出是否满足需求"有了更清晰的边界——边界越清晰，模型越敢拒绝看起来不充分的内容。结果就是假阳率下降（垃圾更难蒙混过关），但假阴率上升（更多边缘正确的内容被拒绝）。

### 实验 artifact 的讨论

L1-L3 的驳回结果在一定程度上受到产物呈现方式的影响：如果向 Agent B 提供完整的文件内容而非摘录，判定可能不同。但这也引出了一个更一般的问题：

在实际的 Agent 工作流中，审查者（Agent B）看到的同样是 Agent A 的产出——而非人类标注的 ground truth。如果 Agent A 的产出本身在形式和完整性上有波动（这在自动生成中很常见），审查者面临的标准校准问题就是真实存在的，而非仅仅源于实验构造。

## 小结

P1 实验的数据说明：

1. 对抗验证（单 Agent B）可以将 Phase Gate 的 50% 假阳率降至 0%
2. 代价是假阴率从 0% 升至 75%
3. 这个结果与前序实验的精度-召回权衡模式一致——模型越强，精度越高但召回越低
4. 4 个垃圾场景的技术特征表明：代码层面的内容质量检查（长度、格式、关键词）可能覆盖其中大部分，不需要依赖 LLM

下一步（P2）将测试 Agent B 自身的输出稳定性——单次实验的 75% 假阴率是否可重现，还是存在较大的随机波动。
