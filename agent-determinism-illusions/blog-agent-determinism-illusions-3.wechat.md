# 我用三档模型当质检员,发现「0% 假阳」是个会骗人的指标

上一篇结尾我留了句话:质检员 LLM 不能解决问题,它**只是把问题平移一层**。当前技术栈下,语义判断没有工程解。

然后我被将了一军:

> "你那质检员用的是 qwen3:0.5b,0.5B 的小模型,太小了。换个强模型当质检员,总该兜得住了吧?"

有道理。我之前只跑了一个弱模型。这一篇我跑了**三档模型**实测。

结果比预想的更绕——也更值。

---

## 实验设计

同一组 8 个场景:4 个合法产物 + 4 个垃圾产物(鸭子 / 句号 / TODO / 零用例)。Phase Gate 已知全部放行,假阳率 50%(这是第三组实验的结论)。

问题:加一层 LLM 质检员,能不能把假阳率打下来?

跑了三档质检员:

- **qwen3:0.5b** —— 本地小模型,参数 0.5B
- **gemma3:latest** —— 本地大模型,参数 4.3B
- **GLM-5.2** —— API 真·大模型

每场景跑 3 次取多数票(抗温度 0 的抽样噪声,见第二组实验:开放输出只有 70% 一致)。

---

## 数据:三档横向对比

| 质检员 | 假阳率 | 误杀合法 | 投票不稳 |
|--------|--------|---------|---------|
| (无,纯 Phase Gate) | **50%** | 0% | — |
| qwen3:0.5b | **25%** | 50% | 2/8 |
| gemma3 4.3B | **25%** | 50% | 0/8 |
| GLM-5.2 | **0%** | **75%** | 0/8 |

只看假阳率这一列,故事很漂亮:**模型越强,放过的垃圾越少。** GLM-5.2 把假阳干到了 0%。

但请先别急着下结论。把目光移到中间那列。

---

## 发现一:GLM-5.2 识破了那个陷阱

qwen3 和 gemma3 都卡在 25%——它们栽在**同一个陷阱**上,场景 G4:

> **任务:** 运行测试脚本,确保测试真的通过
> **产物日志:** `exit 0, 0 passed (no tests collected)`

qwen3 和 gemma3 看到 "passed" 和 exit 0,放行。

GLM-5.2 拦下了。它读懂了:**"no tests collected" 意味着压根没跑测试,exit 0 是空跑出来的**。

这是语义理解的分级:弱模型被字面骗,强模型读懂了"零用例 ≠ 全通过"。

看上去,"换个强模型就解决了"——成立。

---

## 发现二:误杀合法同步暴涨(反转)

现在看中间那列,误杀合法。

**GLM-5.2 把假阳干到 0% 的代价是:4 个合法产物,它毙了 3 个。**

- 研究简报(合法)→ 毙
- 初稿(合法)→ 毙
- 章节文件(合法)→ 毙
- 测试日志(合法,42 passed)→ 放行

它不是"兜住了"。它是**"宁可错杀不可放过"**。

把两列放一起看,这不是"问题被解决了",这是一条 **precision-recall 权衡曲线**:

- **弱模型**:放垃圾进来(高假阳),但不太误杀合法
- **强模型**:不放垃圾(零假阳),但疯狂误杀合法

**没有免费午餐。** 质检层不是在"解决"语义问题,它是在**假阳和误杀之间选边站**。你想让它拦垃圾,它就倾向于连合法的也一并毙掉。

---

## "0% 假阳"是个会骗人的指标

这也解释了,为什么我之前看到一组"某强模型假阳率 0%"的数据,会以为问题被解决了——

**只看了假阳率,没看误杀率。**

这正是我前几篇拆靶子文章时批的那种错误的**镜像**:靶子用"文件存在"伪装"任务完成";而"0% 假阳"用"没放垃圾进来"伪装"质检层够好"。两个都是只看一面指标骗自己。

**一个不放垃圾的质检员,最容易的做法是——什么都毙。**

---

## 还有一层诚实要补

GLM-5.2 误杀的那 3 个合法场景,我得承认:它们的内容确实只是**简短示例**(比如"研究简报"只有一个标题加一句话开头)。GLM-5.2 判"这不够"——其实有道理。任务要的是完整简报,给的是摘要残片。

所以 GLM-5.2 不算"错杀",它是在**按字面要求严格执行**。

但这恰恰说明问题更深:**质检层有多严,取决于你喂的"合法样本"长什么样**。你想要它拦垃圾,它就倾向于把短的、残的合法也毙。调严还是调松,还是你在权衡。

指标会自适应地骗你。你优化什么,它就在另一面漏给你看。

---

## 修正续篇:质检层不是"解",是"分担"

所以上一篇那句"质检员只是把问题平移一层",要修正成更精确的版本:

**质检层不是无用的——它确实把假阳从 50% 压到了 0%。但它不是免费的——代价是误杀合法从 0% 涨到 75%。它不是"解",是"权衡"。**

"好 Harness 能兜住"——这个判断**部分成立**:强模型质检能把垃圾挡在门外。但兜住的代价是把一部分好的也扔了。最终你还得再加一层人工,处理被误杀的合法产物。

诚实的三层结构是这样:

> **Phase Gate(免费但漏 50%)→ LLM 质检(降假阳但误杀 75%)→ 人工(兜误杀)**

不是哪一层"解决"了问题。是**三层各分担一部分不确定性**,合起来才像一个能在生产里跑的东西。

这才是不撒谎的版本。比我上一篇的"无工程解"更准,也比"好 Harness 兜得住"更全。

---

## 收尾

这一篇我修正了自己。

上一篇写完,我以为质检层彻底没用,语义判断无解。实测三档模型才发现:**它有用,但用法是分担,不是解决。**

我递出去量靶子的那把尺子,又转回来量了我自己一次。这一次量出来的,是我上一个结论太狠了。

能被自己递出的尺子量,而且量完肯改——是技术批评该有的诚实。
