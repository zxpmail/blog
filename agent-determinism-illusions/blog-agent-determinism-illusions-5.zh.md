# 六个对抗验证实验——75% 那堵墙没动过

**Agent Determinism Illusions (Part 5)**

---

> **一句话论点：** 审查官是一条画线的机制。每次修复都在移动这条线——但这根线无法被消除，因为它生活在一个三维曲面上，多个合理的边界在此相交。所以 75% 假阴率的墙没有动过，务实的做法是停止试图移动它。

---

## 1. 那堵墙

设定很简单。让一个 LLM 审查 AI agent 的输出，判断是否满足任务要求。输出中混杂着明显的垃圾（"我是一只小鸭子，嘎嘎嘎"、"。"、TODO 占位符、零条收集的测试）和合法成果（研究简报、草稿文档、通过测试的运行结果、代码、翻译）。第一轮 8 个场景，第二轮扩展到 30 个。

当审查官足够敏锐、能抓住所有垃圾时，它停在 0% 假阳率和 75% 假阴率——四分之三的有效输出被拒绝。这就是那堵墙。GLM-5.2 和 deepseek-v4-flash 都撞上了它。较小的模型（qwen3:0.5b、gemma3:4.3b）在曲线上更早的位置——25% / 50%——放走了一些垃圾，拒绝了较少的有效输出。它们不是更好，只是在同一根曲线上的不同工作点。

我尝试了三种标准手段来移开这堵墙。

**重跑并多数投票同一 prompt。** 每场景 N=10 次重跑。在有效调用足够多的每个场景上，判决都是一致的。75% 是系统性的，不是随机的——模型每次都做出了同样的错误判断。对一个不变的判决，你投不出不同结果。

**跨不同 prompt 投票。** 以严格、平衡和宽松三种 prompt 分别评判每个场景。分裂投票是一个有用的信号——它标记出测试集本身就存在争议的场景。但多数投票仍然停在 75% 假阴率，因为三个 prompt 共享相同的偏置方向。为什么？第二节的回答：模型的边界是稳定的；prompt 措辞标记了这条线，并没有移动它。投票可以平滑噪声，不能消除偏置。

**校准 prompt 措辞。** 一个"平衡"prompt（v3）在 8 个 Phase Gate 场景上达到了 100% 准确率。标准的"校准你的 prompt"建议似乎奏效了。扩展到 30 个场景后，v3 和严格版 v2 在所有有效调用上输出了完全一致的判决。8 个场景上的改善只是测试集构成偏差——原始场景恰好有利于 v3 的宽松倾向。

这堵墙是真的。标准杠杆没有一个移动了它。

## 2. 为什么墙不动

审查官是一条画线的机制。这条线区分"充分的输出"和"不充分的输出"——这就是全部工作。形式化检查、LLM 判断、prompt 措辞——这些都是在哪里、怎么画线的选择。

这里是一个关键性质：一条更锋利的线抓住更多垃圾，也会拒绝更多边缘但有效的输出。同样的锋利度，对两种错误类型有相反的效果。把线画得更锋利，假阳率下降而假阴率上升。变钝则相反。精确率-召回率的权衡不是模型的缺陷——它是用不完美的分辨力画线的几何必然。一个完美的审查官不会有这个权衡；审查官对边界的位置有意见，而这些意见是有噪声的。

六个实验是三次画一条更好线的尝试。Phase Gate 在形式上画线——文件是否存在、退出码是否为 0——这与内容无关。四块垃圾（"我是一只小鸭子"、"。"、TODO 占位符、零条收集的测试）轻松通过。假阳率：50%。对抗验证用 LLM 在语义上画线。锋利得多。抓住了所有垃圾（假阳率→0%），同时同样的锋利度拒绝了四分之三的边缘但有效的输出（假阴率→75%）。Prompt 校准试图通过改变措辞来移动线——严格 vs. 平衡 vs. 宽松。在 30 个场景上，v2 和 v3 在所有有效调用上输出了完全一致的判决。线没有移动，因为措辞并不画线。措辞标记线。第三次尝试是替换方法的极限：一旦你在用词语来移动模型已经画好的线，你不再是在替换——你只是在装饰。

那么为什么不用一条更锋利的线——或者一种不同类型的线——既能抓住垃圾又不牺牲有效输出？因为这条线并不生活在一维空间里。

"充分"和"不充分"的边界至少取决于三个独立的问题。谁消费输出——一个全盘接受的新手工程师，还是一个会抓住边缘情况的资深审阅者？部署在哪里——下周就扔掉的原型，还是运行多年的生产系统？出错了会怎样——一个在会上丢脸的演示，还是一个导致服务宕机的部署？

这三个维度大致独立。它们有相关性——消费者类型给部署上下文一个弱提示——但不足以坍缩成一个轴。知道消费者并不能确定部署。知道部署并不能确定失败成本。所以边界不是一维空间中的一个点；它是一个三维空间中的曲面。而大多数真实输出落在内部——多个合理边界交叉的地方。

"这个输出够不够？"没有一个单一答案，因为问题本身缺少限定条件。不同的消费者、上下文和成本给出不同的合理答案。这种模糊性不是弱模型的属性——它是问题的属性。

实用结论来自这个几何结构。如果模糊性在问题本身，没有模型能消除它。没有 prompt 能消除它。没有投票方案能消除它。它们只是在同一个曲面的不同位置画线。75% 没有在四个模型之间移动，因为无处可移——沿着曲面移动工作点只是在 FP 和 FN 之间做交换，曲面本身不会消失。

我们不是没有找到正确的技巧。我们在找一种不存在的技巧。

## 3. 围绕墙做设计

那么就围绕它设计。关键动作不是"修墙"——而是"停止试图修墙"。这种接受会改变设计。

如果 75% 是结构性的，你就停止在规则能抓住的垃圾上浪费 LLM 调用（关键词匹配抓住"我是一只小鸭子"，长度检查抓住"。"）。你停止试图投票走出系统性偏置。你停止校准 prompt 措辞并假装模型的边界会跟随。取而代之的是：在规则管用的地方放规则，一个校准过的 LLM 用在语义真正重要的地方，人和机器在三维边界曲面变得模糊的地方协作——这正是第二节的维度论证告诉你模型产生分歧的位置。实践中：廉价的确定性检查（长度、关键词、格式）抓住明显垃圾，一个校准过的 LLM 调用按需求逐条判断语义剩余，任何分裂判决升级给人。LLM 永远不会看到规则能处理的情况——它只看到规则处理不了的。

然后你选择在墙的哪一边。这不是一个待办事项；它是其余设计所依赖的决策。更多假阳性意味着更多审阅注意力被浪费在被标记为可疑的有效输出上。更多假阴性意味着更多有缺陷的输出流入生产。这个权衡是结构性的。唯一的错误是假装你不必选择。

## 4. 幻觉一直在移动

这个系列叫"Agent Determinism Illusions"。六个实验下来，幻觉一直在移动。

它始于输出确定性——temp=0 本该保证一致性，但它没有（结构化任务上同一个 listing 的 20 个不同版本）。被抓住后，幻觉移到了审查标准——形式化检查本该保证质量，但它们没有（"文件存在"通过了"我是一只小鸭子，嘎嘎嘎"）。再次被抓住，它移到了解决方案的复杂度——多模型投票、校准过的 prompt、分层管线总该管用吧。它们并没有真正管用；每一层继承同一堵墙。第三次被抓住后，幻觉不再藏在技术假设里，而是上升了一个层次：进入元期望——足够的实验应该产生一个干净的结论。它们产生的结论是：没有干净结论。

幻觉之所以一直在移动，是因为我们一直在追。工作不是抓住它——工作是停止期望它静止不动。
