## 三层决策表

| Layer | 输入 | 逻辑 | 输出 | Cost | Phase 2 检测率 |
|-------|------|------|------|------|:---:|
| **L0 证据门** | 文件系统 | 存在且非空 | pass/fail + 缺失清单 | ~0ms | **66.7%**① |
| **L1 合约正则** | 合约 + 证据内容 | 逐需求 regex 匹配 | pass/fail + 不符清单 | ~0ms | **80.0%** |
| **L2 逐需求 LLM** | 合约 + 证据内容 | 逐需求 LLM 判断 | pass/fail + 驳回理由 | ~1s/REQ | **100%** |

① L0 在 Phase 1 的检测率（9 非合规场景）。它的核心价值不是检测率，而是**确定性：无模型、无方差、不欺骗**。

## 分流逻辑

```
IF 有合约 THEN
    L0 → (pass?) → L1 → (pass?) → L2 → DONE/REJECT
ELSE
    L0 → (pass?) → ChA LLM → DONE/REJECT
END

任意层 REJECT → 终止并返回驳回理由
```

无合约时降级到自由文本 LLM（Phase 1 的 Channel A），因为 C1/C2 需要合约定义才能工作。

## 各层的盲区互补

```
盲区类型              L0       L1       L2
                    证据门    合约正则   逐需求LLM
─────────────────────────────────────────────
证据完全缺失           ✅ 检出   ✅ 检出    ✅ 检出
空证据文件             ✅ 检出   ✅ 检出    ✅ 检出
部分证据缺失           ✅ 检出   ✅ 检出    ✅ 检出
伪造证据内容           ❌ 漏放   ❌ 漏放①   ⚠️ 50%②
过期证据               ❌ 漏放   ❌ 漏放    ⚠️ 可能检出
语义缺口 (DPI)         ❌ 漏放   ❌ 漏放③   ✅ 检出
agent 话术说服         ❌ 漏放   ✅ 免疫    ✅ 免疫
命名不一致             ❌ 漏放   ⚠️ 看正则宽窄 ✅ 理解同义词
数值约束              ❌ 漏放   ✅ 精确     ✅ 精确
```

① L1 在 SC14 中正确拒绝了伪造（1/5 正则匹配），因为伪造内容太泛泛
② L2 在 SC14 中 2/5 拒绝——部分被说服
③ L1 在 SC10a 中误把 "not write-invalidation" 当成匹配

## 合约设计指南

由实验可以总结合约正则（C1）的可靠性分层：

| 约束类型 | 可靠性 | 示例 | 说明 |
|---------|:-----:|------|------|
| 数值比较 | ✅ 高 | `(8[5-9]|90+)%` | 免于语义模糊 |
| 转义模式 | ✅ 高 | `0\s+errors` | 格式稳定 |
| 功能名称 | ⚠️ 中 | `write.?invalidat` | 命名一致性敏感 |
| 否定文本 | ❌ 低 | `not write-invalidation` 误匹配 | 需要负向前瞻 |
| 存在性 | ✅ 高 | 合约层不直接使用，L0 已覆盖 | — |

## 与 ReqForge content-verify 的关系

```
ReqForge L0e (Re-Stat) ──── 输出文本启发式（未来时态/桩代码/注释比）
                                  │ 文本通道，受 DPI 约束
                                  │ 替代方案：L1 合约正则（文件系统通道）
                                  ▼
本 Pipeline ──── L0 证据门 → L1 合约正则 → L2 逐需求 LLM
                  │            │              │
                  文件系统通道   合约定义       合约 LLM
                  ╰── 确定性 ──╯  ╰── 补 DPI ─╯
```

ReqForge 的 L0e 和本 Pipeline 的 L1 合约正则都是**不需要模型的过滤器**。区别是 L0e 在**文本通道**做模式匹配（受 DPI 约束），L1 在**文件系统通道**做模式匹配（不受 DPI 约束）。后者更可靠，但前提是有明确的合约定义。
