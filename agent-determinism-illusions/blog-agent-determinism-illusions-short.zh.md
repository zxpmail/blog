# 朋友圈 / 转发语版(~450 字)

刷到一篇 7000 行的生产级 Agent 文章,核心主张是"在 LLM 不确定性上叠加确定性约束"。方向对,但我跑了三组实验,它三个"确定性"支柱全崩:

**一、词汇重叠度判新旧任务?50% 误判。**
文章说 ≥0.24 判同任务、≤0.08 判新任务、"80% 秒判"。
实测:"继续写那篇循环引擎的文章" vs "删掉那篇循环引擎的文章" → 重叠度 0.615,判为**同任务**——用户说删,引擎继续写。
"修复支付 bug" vs "结账页报错了看看" → 重叠度 0.000,判为**新任务**。同义改写、跨语言全错。代码敢判的 27 例里只对 12 例,准确率 44%。

**二、温度 0 = 确定性?开放输出只有 70% 一致。**
文章说评估器温度 0 "几乎完全确定"。
同一 prompt 跑 20 次:数学 100%、列举 95%、开放创意 **70%**——冒出 5 个不同版本,最低相似度 0.198("Always head Northbound..." vs "Premium coffee for the journey ahead")。评估器输出的 done/reason/evidence 正是这类开放文本,本质不稳定。唯一 100% 的是 17×23=391。

**三、Phase Gate = 任务完成?假阳率 50%。**
文章说"把任务完成变成可验证的客观事实"。
实测:研究简报内容是"我是一只小鸭子"——✅通过;初稿只有一个句号——✅通过;3 个章节文件全是"TODO"——✅通过;测试"0 passed (no tests collected)" exit 0——✅通过。
Gate 只验证"动作发生了",验证不了"结果是对的"。中间那道鸿沟叫内容质量。

**结论:** 文章 7000 行 Rust 大概率真,但守的全是符号层(字符串、文件路径、exit code)的门,语义层(意图、内容、质量)裸奔。问题意识值钱,具体答案别照搬。
