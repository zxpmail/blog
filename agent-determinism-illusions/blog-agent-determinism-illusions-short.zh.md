# 朋友圈 / 转发语版(~450 字)

刷到一篇 7000 行的生产级 Agent 文章,核心主张是"在 LLM 不确定性上叠加确定性约束"。方向对,但我跑了**四组**实验,它的"确定性"全崩、我的方案也不行:

**一、词汇重叠度判新旧任务?50% 误判。**
文章说 ≥0.24 判同任务、≤0.08 判新任务、"80% 秒判"。
实测:"继续写那篇循环引擎的文章" vs "删掉那篇循环引擎的文章" → 重叠度 0.615,判为**同任务**——用户说删,引擎继续写。
"修复支付 bug" vs "结账页报错了看看" → 重叠度 0.000,判为**新任务**。同义改写、跨语言全错。代码敢判的 27 例里只对 12 例,准确率 44%。

**二、温度 0 = 确定性?开放输出只有 70% 一致。**
文章说评估器温度 0 "几乎完全确定"。
同一 prompt 跑 20 次:数学 100%、列举 95%、开放创意 **70%**——冒出 5 个不同版本,最低相似度 0.198。评估器输出的 done/reason/evidence 正是这类开放文本,本质不稳定。唯一 100% 的是 17×23=391。

**三、Phase Gate = 任务完成?假阳率 50%。**
文章说"把任务完成变成可验证的客观事实"。
实测:研究简报内容是"我是一只小鸭子"——✅通过;初稿只有一个句号——✅通过;3 个章节文件全是"TODO"——✅通过;测试"0 passed (no tests collected)" exit 0——✅通过。
Gate 只验证"动作发生了",验证不了"结果是对的"。中间那道鸿沟叫内容质量。

**四、我自己也撒谎了——embedding 一样不行。**
我准备了三组实验后用"embedding 升级词汇重叠,多票修补评估,质检员补充 Gate"来当建设方案。
然后跑了个验证:Qwen3-embedding:0.6b 跑同义/反义/无关分离。结果:同义 0.766,反义 0.739——**只差 0.026**。"别优化代码性能"和"优化代码性能"余弦 0.881,超过 10/12 个同义对。
embedding 能分"相关 vs 无关",但**不能分"同方向 vs 反方向"**。当话题相同时,同向和反向的向量几乎一样。我那套"升级"方案在这组数据前不成立。

**真实结论:** 当前技术栈下,当用户在当前话题下说了方向不明的插话,工程上不应该硬判。检测到话题重叠后,直接问人。LLM 做符号层操作,人覆盖语义判断——不性感,但不撒谎。
