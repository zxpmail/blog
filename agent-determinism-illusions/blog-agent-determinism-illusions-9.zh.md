# 我的"调 prompt 就行"只通过了 8 个样本

## 30 个样本之后，校准效果消失了

**Agent 确定性幻觉（续篇八，本系列终章）**

## 前情

[上一篇文章](blog-agent-determinism-illusions-8.zh.md)记录了一个乐观的发现：在 8 个 Phase Gate 场景上，"公正"prompt（v3）达到了 100% 准确率，把"严格"prompt（v2，P1 基线）的假阴率从 75% 降到 0%。结论是"对抗验证的解法不在更强的模型或更复杂的投票机制，而在校准审查指令的严格度"。

这篇文章是那个结论的反驳。需要说明的是，[上一篇文章](blog-agent-determinism-illusions-8.zh.md)在"实验局限"一节中已经声明了"结论是指示性的，不是结论性的"，并预告了下文的扩展检验——它不是无条件的定论，而是有条件的中期报告。本文的数据是那个中期报告的后续验证。

## 实验 P4：扩大测试集

我把测试集从 8 个场景扩展到 30 个，覆盖更多任务类型（代码生成、翻译、邮件写作、参数解析、命令执行），增加了"边缘"类别（方向正确但质量不足、格式正确但内容不够）。然后拿 v2（严格）和 v3（公正）两个 prompt 各跑一轮。

30 个场景 × 2 prompt = 60 次调用。

### 结果

| 标注方案 | v2 严格 ACC | v3 公正 ACC |
|---------|-------------|-------------|
| 排除边缘（20 样本） | 88.2% [66-97%] | 88.2% [66-97%] |
| 全部，边缘标正确 | 66.7% [47-82%] | 73.9% [54-87%] |
| 全部，边缘标垃圾 | 87.5% [69-96%] | 82.6% [63-93%] |

所有置信区间重叠。两个 prompt 在统计上没有差异。

更直接的数字：**v2 和 v3 在有效调用上 100% 一致**——21/21 个双方都返回有效结果的场景，判定完全相同。

前文的"校准改善"没有复现。

### 发生了什么

P3b 的 8 个测试集场景以 Phase Gate 场景为主（研究简报、初稿、章节文件），v3 对这些摘录风格的输出确实比 v2 更宽松一些。但当测试集扩展到代码、翻译、邮件、解析、表格等多种任务类型后，这种差异消失了。

原因：**模型的审查行为主要由任务结构和输出格式决定，而非 prompt 措辞。** "搜索严格"和"公平判断"两个指令在模型推理中产生了相同的判定规则。C02-C03（给摘要而非完整文件的场景）被两个 prompt 一致驳回。

这解释了为什么 P3b 的"改善"没有泛化——它测量的不是 prompt 差异，而是测试集组合偏差。

## 回顾：这个系列真正走了多远

从 P0 到 P4，六轮实验，六条未走通的路径：

- **温度 0 不保证一致。** 自由文本输出在 temp=0 下发散。但这不是主要矛盾——结构化二值判定的稳定性很高。
- **对抗验证消灭了假阳。** 但产生了 75% 的假阴。精度-召回权衡是结构性约束。
- **系统性偏差不能靠采样解决。** 审查官每次做同一个判定，投票没有帮助。
- **多视角分歧是有效信号。** 分裂指示不确定性，但多数决不能纠正偏差方向。
- **Prompt 校准在小测试集上有效。** 但扩大测试集后效果消失——模型的判定规则比预期的更稳定，不受措辞影响。
- **JSON 格式失败率约 15-20%。** 这是使用 LLM 输出做自动化决策的工程门槛。

六条路径的共同点：**每次我们以为找到了解法，都在更大的测试集或更严格的检验下站不住。**

## 真正学到了什么

我不认为这六轮实验没有产出。它们产出了四张可用的地图。

**第一张：精度-召回权衡的分布。**

这个系列跑了约 260 次 API 调用，覆盖了多种任务类型。跨所有实验，假阳率容易控制（垃圾场景特征明显），假阴率在不同场景间差异很大（完整文件容易通过，摘录类输出容易被驳回）。这意味着误差不是均匀分布的——它是**场景依赖的**。知道哪些场景容易翻车，比知道一个平均准确率更有用。

**第二张：审查官的一致性边界。**

P2（N=10 一致）和 P4（v2/v3 100% 一致）共同指向一个结论：**结构化审查判定是高度确定的。** 模型对同一输入不会在二值判定上随机波动。这既是好消息（可预测），也是坏消息（系统性偏差不会自己消失）。

**第三张：JSON 作为接口时的可靠性窗。**

跨实验 15-20% 的调用失败率是一个工程现实。如果你依赖 LLM JSON 输出来驱动自动化，必须内置重试和 fallback。

**第四张：标注本身的模糊区。**

30 个场景中约 1/3 属于"边缘"类别。这部分场景的标注可以争论两个方向，而换标注方案会使准确率变化 5-10 个百分点。这意味着测试集本身的质量上限受限于标注一致性——而不是模型能力。

## 如果这次是真的结束

系列标题叫"Agent 确定性幻觉"。六轮实验后，这个词的含义变了四次：

第一次，它指 LLM 输出不一致的幻觉——人们以为 temp=0 能保证确定。

第二次，它指审查标准的幻觉——人们以为形式检查能保证内容质量。

第三次，它指解决方案的幻觉——人们以为需要复杂的架构来解决一个 prompt 工程问题。

第四次，它指**终结方案的幻觉**——人们以为足够多的实验就能找到一个干净的答案。

没有干净的答案。精度-召回权衡是审查任务的结构性约束。Prompt 校准在边际上有用，但不能消除这个约束。多视角投票提供额外信号，但不能替代根本的校准问题。更大的测试集让结论更稳健，但也揭示了更多的边缘情况。

这些不是失败——它们是把"我们不知道的"变成"我们知道我们不知道的"的过程。

如果你只带走一件事，是这个：

**对抗验证的有效性不取决于你用什么模型，也不取决于你怎么写 prompt，而取决于你是否接受"没有完美方案"这个前提。** 精度-召回权衡不是你可以绕过的——它是你必须在假阳和假阴之间选择一个可接受的比例。

一旦你接受了这个前提，选择就清楚了：用简单规则覆盖明显垃圾，用校准后的 LLM 做第二层，剩下的分裂场景升等人审。接受剩余误差，而不是试图消除它。
